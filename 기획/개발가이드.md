# 개발 가이드

## 안전진단 — 데이터 소스 분류

안전진단 기능은 "외부 API로 가져올 수 있는 데이터"와 "사용자가 직접 입력해야 하는 데이터"로 나뉜다.

---

### 전세안전진단

#### 사용자 직접 입력
| 항목 | 이유 |
|------|------|
| 전세 보증금 | 계약 협의 중인 금액이므로 외부에서 알 수 없음 |
| 선순위 근저당 | 등기부등본을 직접 열람해야 확인 가능. 공개 API 없음 |
| 주택 유형 | 아파트 / 빌라·다세대 / 단독주택 / 오피스텔 선택 |
| 전용 면적 | 실거래가 조회 시 필터 기준으로 활용 |

#### 외부 API 연동 가능
| 항목 | API / 출처 | 비고 |
|------|-----------|------|
| 추정 시세 | 국토교통부 실거래가 API (data.go.kr) | 아파트: `getRTMSDataSvcAptTrade`, 연립·다세대: `getRTMSDataSvcRHTrade` |
| 공시가격 | 공동주택 공시가격 API (data.go.kr) | HUG 보증 한도 계산 시 활용 |
| 건축물대장 | 건축물대장 정보 서비스 API (data.go.kr) | 위반건축물 여부, 용도 확인 |
| 주소 → 법정동 코드 | 카카오 주소 API (이미 연동됨) | 실거래가 API 조회 시 필요 |

#### 계산으로 산출 (API 불필요)
| 항목 | 계산식 |
|------|--------|
| 전세가율 | 보증금 ÷ 추정 시세 × 100 |
| 부채비율 | (선순위 근저당 + 보증금) ÷ 추정 시세 × 100 |
| HUG 보증 가능 여부 | 부채비율 ≤ 90% (다세대: 공시가격 × 140% × 90% 기준 병행 적용) |
| HF 보증 가능 여부 | 부채비율 ≤ 100% (수도권 기준, 주택 유형별 상이) |

#### 외부 링크 유도 (API 없음, 사용자가 직접 확인)
| 항목 | 이동할 사이트 |
|------|-------------|
| 등기부등본 확인 | 대법원 인터넷등기소 (www.iros.go.kr) |
| 임대인 체납세금 | 국세청 미납국세 열람 신청 (임대인 동의 필요) |
| 전세보증보험 상세 | HUG 안심전세 (khug.or.kr) |

### 매매안전진단

#### 사용자 직접 입력
| 항목 | 이유 |
|------|------|
| 매매 희망가 | 협의 중인 가격이며 실거래가와 다를 수 있음 |
| 주택 유형 | LTV·취득세율 산정 기준 |
| 현재 보유 주택 수 | LTV 등급(무주택/1주택/다주택) 결정 |
| 연소득 | DSR 계산에 필요. 금융기관 외 외부에서 알 수 없음 |

#### 외부 API 연동 가능
| 항목 | API / 출처 | 비고 |
|------|-----------|------|
| 최근 실거래가 | 국토교통부 실거래가 API (data.go.kr) | 가격 적정성 비교에 활용 |
| 규제지역 여부 | 국토교통부 토지이용규제 서비스 또는 정적 데이터 관리 | 투기과열지구·조정대상지역 여부 → LTV 결정 |
| 공시가격 | 공동주택 공시가격 API | 취득세 기준가 확인 |
| 주소 → 시군구 코드 | 카카오 주소 API | 규제지역 매핑에 사용 |

#### 계산으로 산출 (API 불필요)
| 항목 | 계산식 / 기준 |
|------|--------------|
| LTV 한도 | 규제지역·주택 보유 수·주택 유형 조합 → 정부 기준표 적용 |
| 최대 대출 가능액 | 매매가 × LTV 한도 |
| 필요 자기자본 | 매매가 + 취득비용 합계 - 대출 가능액 |
| DSR | (연 원리금 상환액 ÷ 연소득) × 100 (기준: 40% 이하) |
| 월 상환액 | 원리금균등상환 공식, 금리 및 기간 입력 기반 |
| 취득세 | 주택 보유 수 × 매매가 기준 세율 적용 (1주택 1~3%, 다주택 중과) |
| 중개보수 | 매매가 구간별 법정 요율 적용 |

#### 외부 링크 유도
| 항목 | 이동할 사이트 |
|------|-------------|
| 등기부등본 확인 | 대법원 인터넷등기소 |
| 건축물대장 확인 | 정부24 |
| 재개발·재건축 여부 | 토지이용계획 열람 (eum.go.kr) |

## 외부 API 발급 정보

| API | 발급처 | 무료 여부 | 비고 |
|-----|--------|---------|------|
| 국토교통부 실거래가 | data.go.kr | 무료 | 일별 트래픽 제한 있음 |
| 공동주택 공시가격 | data.go.kr | 무료 | 공동주택만 해당 |
| 건축물대장 정보 | data.go.kr | 무료 | |

> 국토교통부 API는 단지명 + 법정동 기반 조회이므로, 주소 입력 → 카카오 API로 법정동 코드 추출 → 실거래가 API 조회 순으로 처리한다.

## 구현 단계

### Phase 1 — 계산 기반 (현재 목표)
- 시세는 사용자 직접 입력 + 국토교통부 API 자동 조회 병행
- 주소 입력: 카카오 주소 검색 팝업 연동
- LTV·DSR·취득세·전세가율·HUG 가능 여부 클라이언트 계산
- API로 가져온 값에는 `?` 아이콘 + 출처 툴팁 표시
- LTV·규제지역 기준은 어드민 설정값으로 관리 (배포 없이 변경 가능)

### Phase 2 — API 고도화
- 건축물대장 API 연동 → 위반건축물 자동 경고
- 공시가격 API 연동 → HUG 보증 한도 계산 정밀화

## 주소 입력 처리 흐름

```
사용자 주소 입력 (카카오 주소 검색 팝업)
  → 카카오 API: 주소 → 법정동 코드 + 시군구 코드
  → 백엔드 프록시 → 국토교통부 실거래가 API
  → 최근 실거래가 반환 (단지명 + 면적 기준 최근 3개월 평균)
```

> data.go.kr API 키는 백엔드에서만 보관. 프론트는 백엔드 프록시 엔드포인트만 호출.

## 진단 결과 저장 (백엔드 API 필요)

| 엔드포인트 | 설명 |
|-----------|------|
| `POST /api/v1/diagnosis` | 진단 결과 저장 (타입: jeonse/maemae/monthly) |
| `GET /api/v1/diagnosis/latest?type=jeonse` | 마지막 진단 결과 조회 |

- 비로그인: 저장 없음
- 로그인 시: 타입별 마지막 1건만 저장 (이력 관리 불필요)

## 기술 스택 현황

| 항목 | 내용 |
|------|------|
| 프론트엔드 | React 18 + TypeScript + Vite + Tailwind CSS v4 |
| 백엔드 | Spring Boot + H2 (개발) / 추후 MySQL 전환 예정 |
| 인증 | 카카오 OAuth2 redirect + JWT Bearer 토큰 |
| 상태 저장 | 비로그인: localStorage / 로그인: 서버 API |
