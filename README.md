# 🏠 자리톡 단기임대 예약 시스템

## 📖 과제 소개

자리톡의 단기임대 서비스가 오픈되었습니다! 🎉

이제 게스트는 숙소를 예약하고, 호스트는 예약을 관리할 수 있습니다.  
단기임대 예약 시스템의 핵심 기능은 **단기임대 예약 API**입니다.

ReservationController 클래스가 제공하는 `단기임대 예약 API`가 요구사항에 맞게 동작하도록 코드를 작성 또는 수정해주세요.
```
POST /houses/{houseId}/reservations
```

요청 예시)
```bash
curl -X POST http://localhost:8080/houses/1/reservations \
    -H "content-type: application/json" \
    -H "x-account-id: 1" \
    -d '{
        "checkInDate": "2026-03-01",
        "checkOutDate": "2026-03-08"
    }'
```

## ⚙️ 기술 스택

- Java 21
- Spring Boot 3.2.4
- Maven

## 🎯 요구사항

### 1. 예약은 최소 1주일(7일) 이상부터 가능합니다.

체크인 날짜부터 체크아웃 날짜까지의 일수를 기준으로 합니다.

예시)
- 불가: 2026-01-01 ~ 2026-01-06 (6일)
- 가능: 2026-01-01 ~ 2026-01-07 (7일)
- 가능: 2026-01-01 ~ 2026-01-10 (10일)


### 2. 이미 예약된 날짜는 예약할 수 없습니다.

예시)
- 예약된 기간: 2026-01-05 ~ 2026-01-10
- 불가: 2026-01-08 ~ 2026-01-15 (기간이 겹침)
- 가능: 2026-01-10 ~ 2026-01-17 (체크아웃 날짜부터는 예약 가능)

### 3. 호스트가 예약 차단을 설정한 기간에는 예약할 수 없습니다.

예시)
- 차단된 기간: 2026-01-01 ~ 2026-01-03
- 불가: 2026-01-02 ~ 2026-01-09 (차단 기간과 겹침)
- 가능: 2026-01-04 ~ 2026-01-11 (차단 종료일 다음날부터 가능)

### 4. 자신 소유의 숙소는 예약할 수 없습니다.

### 5. 예약 완료 시, 알림 메세지를 발송하고, 이력을 남깁니다.

1. 게스트에게 알림 메시지 발송
   - 실제 발송은 `System.out.println()` 으로 대체합니다.
   - 예시 발송 메세지: "김자리님의 예약이 완료되었습니다!"
2. 알림 이력 저장
   - `NotificationHistory` 테이블에 기록합니다.


## 🗂️ 제공 테이블 설명

| 테이블              | 설명                |
|---------------------|-------------------|
| Account             | 회원 정보             |
| House               | 숙소 정보             |
| BlockedDate         | 호스트가 설정한 예약 차단 기간 |
| NotificationHistory | 알림 발송 이력          |


## 🛠️ 기술 제약사항

본 과제는 비즈니스 로직 구현에 집중하기 위해 인증 처리를 간소화했습니다.

- HTTP 헤더 `x-account-id` 로 사용자를 식별합니다.
- 실제 인증 없이 전달된 값을 신뢰합니다.
