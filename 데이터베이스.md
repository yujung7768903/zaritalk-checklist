# 데이터베이스

## 설계 원칙
- 체크리스트 **항목 내용**은 코드(정적 상수)로 관리 → DB 저장 불필요
- DB에는 **"누가 어떤 항목을 완료했는가"** 만 저장
- 비로그인 User의 진행 상태는 localStorage로만 관리 → DB 불필요

---

## ERD

```mermaid
erDiagram
    user {
        BIGINT pk PK
        BIGINT kakao_id "UNIQUE"
        VARCHAR nickname
        DATETIME created_at
    }

    checklist_progress {
        BIGINT pk PK
        BIGINT user_pk "user.pk 참조"
        VARCHAR checklist_type "NEW_HOME, MOVE"
        DATETIME created_at
    }

    checklist_item_progress {
        BIGINT pk PK
        BIGINT checklist_progress_pk FK
        VARCHAR item_id "프론트 상수 식별자"
        BOOLEAN completed
    }

    diagnosis {
        BIGINT id PK
        BIGINT user_pk "user.pk 참조"
        VARCHAR type "jeonse, maemae, monthly"
        TEXT input_json
        TEXT result_json
        DATETIME created_at
    }

    user ||--o{ checklist_progress : "보유"
    user ||--o{ diagnosis : "보유"
    checklist_progress ||--o{ checklist_item_progress : "보유"
```

## 테이블 상세

### user
카카오 로그인으로 생성되는 사용자

| 컬럼 | 타입 | 제약 | 설명 |
|------|------|------|------|
| pk | BIGINT | PK, AUTO_INCREMENT | |
| kakao_id | BIGINT | NOT NULL, UNIQUE | 카카오 사용자 ID |
| nickname | VARCHAR(100) | | 카카오 닉네임 |
| created_at | DATETIME | NOT NULL | |

### checklist_progress
User 1명 × 체크리스트 타입 1개 = 행 1개

| 컬럼 | 타입 | 제약 | 설명 |
|------|------|------|------|
| pk | BIGINT | PK, AUTO_INCREMENT | |
| user_pk | BIGINT | NOT NULL | user.pk 참조 (논리적 FK) |
| checklist_type | VARCHAR(20) | NOT NULL | `NEW_HOME` / `MOVE` |
| created_at | DATETIME | NOT NULL | |

- `(user_pk, checklist_type)` UNIQUE

### checklist_item_progress
항목별 완료 여부

| 컬럼 | 타입 | 제약 | 설명 |
|------|------|------|------|
| pk | BIGINT | PK, AUTO_INCREMENT | |
| checklist_progress_pk | BIGINT | NOT NULL, FK | checklist_progress.pk 참조 |
| item_id | VARCHAR(100) | NOT NULL | 프론트 상수와 일치하는 항목 식별자 |
| completed | BOOLEAN | NOT NULL, DEFAULT FALSE | 완료 여부 |

- `(checklist_progress_pk, item_id)` UNIQUE

### diagnosis
안전진단 결과. 유형별 최신 1건만 유지

| 컬럼 | 타입 | 제약 | 설명 |
|------|------|------|------|
| id | BIGINT | PK, AUTO_INCREMENT | |
| user_pk | BIGINT | NOT NULL | user.pk 참조 (논리적 FK) |
| type | VARCHAR(20) | NOT NULL | `jeonse` / `maemae` / `monthly` |
| input_json | TEXT | | 진단 입력값 |
| result_json | TEXT | | 진단 결과 |
| created_at | DATETIME | NOT NULL | |
